# Day 2 - Salesforce Platform Basics

## 1. What is Salesforce Platform?

Salesforce Platform is a cloud-based CRM platform that helps businesses manage customers, sales, services, and applications. It provides tools for building apps, storing data, automating workflows, and extending functionality using configuration and coding.
It follows a multi-tenant architecture where multiple organizations share the same infrastructure securely.

---

# 2. Explain
## App
An App in Salesforce is a collection of tabs, objects, and features grouped together for a specific business purpose.
Example:
- Sales App
- Service App
- College Management App

---

## Object
An Object is like a database table that stores data.
Examples:
- Account
- Contact
- Opportunity

Custom Objects can also be created.
Example:
- Student
- Patient
- Course

---

## Tab
A Tab is the user interface element used to access objects and records in Salesforce.
Example:
- Accounts Tab
- Contacts Tab
- Students Tab

Tabs help users navigate through the application.

---

# 3. Difference: Configuration vs Coding

## Configuration (No Code)
Configuration means building functionality using Salesforce tools without writing code.
Examples:
1. Creating validation rules
2. Building workflows and flows

### When to Use
- Simple automation
- Standard business requirements
- Faster implementation

---

## Coding (Apex)
Coding means using Apex, Lightning Components, or APIs to build advanced functionality.
Examples:
1. Complex business logic
2. Custom integrations with external systems
### When to Use
- Advanced automation
- Complex calculations
- External integrations
- Custom UI functionality
